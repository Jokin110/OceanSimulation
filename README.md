A highly scalable, real-time ocean surface simulation built natively in **C++** and **DirectX 11**. 

Based heavily on Jerry Tessendorf's statistical wave models (JONSWAP spectrum), this project moves the immense mathematical overhead of simulating realistic water entirely to the GPU. By leveraging a **Fast Fourier Transform (FFT)** compute pipeline and dynamic hardware tessellation, the engine generates infinitely tiling, deeply detailed ocean surfaces running at high framerates.

## 🔬 Core Technical Features

### 1. GPGPU Fast Fourier Transform (FFT) Pipeline
The physical simulation is handled via a multi-stage Compute Shader pipeline (`cs_5_0`), orchestrated by the `OceanComputeManager` and `FFTManager`. 
* **Initial Spectrum (`InitialSpectrumCS.hlsl`):** Generates the base frequency domain map utilizing a JONSWAP spectrum combined with Gaussian random variables. It factors in physical variables like wind direction, average wind speed, fetch length and swell.
* **Time Evolution (`TimeEvolutionCS.hlsl`):** Advances the frequency domain data forward in time applying the physical dispersion relation of deep water waves.
* **Inverse FFT (`FFTCS.hlsl`):** An adaptation of a GPU implementation of the Cooley-Tukey FFT algorithm (originally developed for Unity by *[Ivan Pensionerov](https://github.com/gasgiant/FFT-Ocean)*). It was ported and integrated into this native DirectX 11 pipeline to perform horizontal and vertical butterfly passes across the textures, transforming the frequency domain data back into spatial displacement maps (X, Y, Z coordinates).

### 2. Dynamic Hardware Tessellation
To render the vast ocean without crippling the GPU, the mesh is dynamically subdivided based on camera distance.
* **Hull Shader (`OceanSurfaceHS.hlsl`):** Calculates Edge and Inside Tessellation Factors based on the proximity of the patch to the `CameraPosition`. Distant patches remain low-poly, while nearby patches are heavily subdivided.
* **Domain Shader (`OceanSurfaceDS.hlsl`):** Evaluates the dynamically generated vertices, mapping them to world space and immediately sampling the `DisplacementTexture` generated by the compute pipeline to physically deform the mesh.

<pre><code class="language-hlsl">// Snippet from OceanSurfaceHS.hlsl - Distance-based LOD calculation
float CalculateTessFactor(float3 v0, float3 v1)
{
    float3 edgeCenterLocal = (v0 + v1) * 0.5f;
    float3 edgeCenterWorld = mul(float4(edgeCenterLocal, 1), m_WorldMatrix).xyz;
    
    float distanceToCamera = length(m_CameraPosition - edgeCenterWorld);
    
    // Exponentially subdivide as the camera approaches the patch
    return max(1.0f, 64.0f * pow(saturate((m_MaxDistance - distanceToCamera) / (m_MaxDistance - m_MinDistance)), m_TessFactorExponent));
}</code></pre>

### 3. Pixel Shading, Foam Generation & Future Work
The `PixelShader.hlsl` currently utilizes a foundational lighting model, serving as a base for future graphical enhancements.
* **Jacobian Foam Generation:** The compute pipeline calculates the spatial derivatives and Jacobian determinant of the wave deformation. If the Jacobian indicates that the wave crest is folding over itself (peaking), the pixel shader dynamically injects foam.
* **Blinn-Phong & Fog:** The water surface is currently lit using a standard Blinn-Phong reflection model, combined with a basic distance fog to blend the distant ocean geometry into the skybox.
* **Future Work:** Expanding the pixel shader is the primary next step for this project. Planned improvements include implementing realistic subsurface scattering (upwelling), proper Fresnel environment reflections and a more advanced atmospheric scattering model to replace the current fog.

### 4. Real-Time Parameter Injection (ImGui)
The `SceneManager` is hooked into **Dear ImGui**, allowing live manipulation of physical constants and rendering variables directly via constant buffers (CBuffers). You can instantly change wind speeds, foam bias, decay factors and lighting without recompiling shaders.

## 🏗️ Architecture Summary

The C++ backend acts as a robust wrapper for the DirectX 11 API:
* **`D3D11Application`**: Handles window creation (via GLFW), SwapChain initialization, DepthStencil configuration and shader compilation.
* **`Object<...>` Template Framework**: A highly generic rendering interface that dynamically binds the correct Input Layouts, Vertex/Hull/Domain/Pixel Shaders and uniquely structured Constant Buffers for any 3D object in the scene.
* **`TimeManager` & `CameraManager`**: Handles high-resolution time tracking for smooth wave evolution and matrix math (View/Projection) for the fly-cam.
